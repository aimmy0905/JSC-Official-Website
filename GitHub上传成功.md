## 🔄 以后如何更新代码？

当你修改了文件后，使用以下命令上传到GitHub：

### 简单方法（推荐）
```bash
# 进入项目文件夹
cd "/Users/aimmy/Desktop/jsc官网设计"

# 添加所有修改的文件
git add .

# 提交修改（记得改成你的修改说明）
git commit -m "更新了某某功能"

# 推送到GitHub
git push
```

### 详细说明

#### 第1步：添加文件
```bash
git add .
```
**解释：** 这就像把你修改的文件放进"快递箱"

#### 第2步：提交修改
```bash
git commit -m "你的修改说明"
```
**解释：** 这就像给"快递箱"贴上标签，说明里面装的是什么

**示例：**
- `git commit -m "修改了产品价格"`
- `git commit -m "添加了新的产品图片"`
- `git commit -m "优化了移动端显示"`

#### 第3步：推送到GitHub
```bash
git push
```
**解释：** 这就像把"快递箱"寄出去，送到GitHub

## 📝 Git命令速查表

| 命令 | 作用 | 通俗解释 |
|------|------|----------|
| `git status` | 查看修改状态 | 看看哪些文件被改了 |
| `git add .` | 添加所有文件 | 把所有修改放进"快递箱" |
| `git add 文件名` | 添加单个文件 | 只把某个文件放进"快递箱" |
| `git commit -m "说明"` | 提交修改 | 给"快递箱"贴标签 |
| `git push` | 推送到GitHub | 把"快递箱"寄出去 |
| `git pull` | 从GitHub拉取 | 从GitHub收"快递" |
| `git log` | 查看提交历史 | 看看以前寄了哪些"快递" |

## 🎓 什么是Git和GitHub？

### Git（版本控制工具）
**是什么：** 一个记录代码变化的工具
**类比：** 就像游戏的"存档"功能，可以回到之前的任何版本

**好处：**
- 📜 记录每次修改
- 🔄 可以回到任何历史版本
- 👥 多人协作不会乱套

### GitHub（代码托管平台）
**是什么：** 一个存储和分享代码的网站
**类比：** 就像百度网盘，但专门存代码

**好处：**
- ☁️ 云端保存，不怕电脑坏了
- 🌍 可以分享给全世界
- 🤝 方便团队协作
- 📊 自动生成网站（GitHub Pages）

## 🆘 遇到问题？

### 常见问题：

#### Q1: 推送失败怎么办？
**答：** 可能是网络问题，或者需要登录GitHub账号。重试几次通常就好了。

#### Q2: 修改后没有显示？
**答：** 记得先 `git add .` 和 `git commit`，然后 `git push`

#### Q3: 如何删除某个文件？
```bash
git rm 文件名
git commit -m "删除了某个文件"
git push
```

#### Q4: 如何回到之前的版本？
```bash
git log  # 查看历史记录
git checkout 版本号  # 回到某个版本
```
